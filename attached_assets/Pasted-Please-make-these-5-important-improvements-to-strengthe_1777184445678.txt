IMPROVEMENT 4 — LOCAL EVENTS SIGNAL
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Add a simulated local events service:

Create: artifacts/api-server/src/services/
eventsSimulator.ts

Logic:
- Generate a random boolean (40% chance of event)
- If true, randomly pick from this list:
  ["Stuttgarter Weindorf", "Schlossplatz Concert", 
   "Stuttgart Marathon", "Weihnachtsmarkt", 
   "Frühlingsfest"]
- Return:
{
  hasNearbyEvent: boolean,
  eventName: string | null,
  expectedFootfall: "high" | "normal" | "low",
  distanceMeters: number (random 100-500)
}

Add this to the GET /api/context response:
events: { hasNearbyEvent, eventName, 
          expectedFootfall, distanceMeters }

In the consumer view context strip, add a 4th 
chip when hasNearbyEvent is true:
"📅 [eventName]" with a purple dot

In the Gemini offer generation prompt, include 
the events data so AI can reference nearby 
events in the offer copy when relevant.

IMPROVEMENT 5 — OFFER EXPIRY FLOW
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

When the countdown timer reaches 0:
- Main offer card background changes to 
  light gray (#f0f0f0)
- Discount pill changes to gray and shows 
  "Expired" instead of percentage
- Headline changes to "This offer has expired"
- Subtext changes to "Fresh offers are generated 
  every few minutes based on your context"
- Show a new teal button: "🔄 Generate new offer"
  which calls POST /api/offers/generate again
- Small text below: "Offer valid for X minutes 
  from generation time"

This shows judges that offer lifecycle is 
intentional and complete.
