WHAT TO BUILD
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

An AI-powered city wallet app that:
1. Fetches real-time context (weather, time, simulated merchant demand)
2. Uses Google Gemini API to dynamically generate personalized local offers
3. Lets users redeem offers via QR code with simulated checkout
4. Shows merchants a live dashboard with performance metrics

Demo scenario:
- City: Stuttgart, Germany
- User: Mia, walking in Altstadt at lunch
- Context: 9°C + overcast + nearby café has low transaction volume
- AI generates: "Cold outside? Your cappuccino awaits — 15% off"

--- weatherService.js ---
- Fetch from OpenWeatherMap:
  URL: https://api.openweathermap.org/data/2.5/weather
       ?lat=48.7758&lon=9.1829&appid=${OPENWEATHER_API_KEY}&units=metric
- Return: { temp, condition, feelsLike }
- If API fails, return fallback: { temp: 9, condition: "overcast", feelsLike: 6 }

--- payoneSimulator.js ---
- Simulate merchant transaction density
- Generate random volume 20–80 for "cafe-muller"
- If volume < 60: status = "low", else "normal"
- Return: { merchantId, volume, status }

--- geminiService.js ---
- Import: import { GoogleGenerativeAI } from "@google/generative-ai"
- Initialize: const genAI = new GoogleGenerativeAI(process.env.GEMINI_API_KEY)
- Use model: gemini-1.5-flash
- Function generateOffer(contextState, merchantRules):

  Send this prompt to Gemini:
  "You are an AI offer generator for a city wallet app.
   Generate a hyper-personalized offer based on this context.
   Respond ONLY with a valid JSON object, no markdown, no explanation.
   
   Context: ${JSON.stringify(contextState)}
   Merchant rules: ${JSON.stringify(merchantRules)}
   
   Return exactly this JSON structure:
   {
     headline: (max 8 words, emotional and situational),
     subtext: (max 12 words, explains why this offer right now),
     discountPercent: (number, within merchant maxDiscountPercent),
     expiryMinutes: (number between 10-25),
     tone: (warm or urgent or playful),
     ctaText: (max 4 words)
   }"

- Parse response text as JSON
- If parse fails, return fallback offer:
  { headline: "Warm up with a cappuccino", subtext: "Cold outside, quiet café nearby",
    discountPercent: 15, expiryMinutes: 20, tone: "warm", ctaText: "Claim offer" }

--- qrService.js ---
- Import qrcode and uuid packages
- Function generateQR(offerId, userId):
  - Create token string: offerId + "_" + uuid()
  - Generate QR as base64 data URL using qrcode.toDataURL(token)
  - Return: { token, qrDataUrl }

BACKEND: ROUTES
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

GET /api/context:
- Call weatherService + payoneSimulator
- Return composite context:
{
  weather: { temp, condition, feelsLike },
  location: { city: "Stuttgart", district: "Altstadt" },
  time: { hour: new Date().getHours(), period: "lunch" },
  merchantDemand: { merchantId: "cafe-muller", volume, status },
  userBehavior: "browsing"
}

POST /api/offers/generate:
- Get context from context route internally
- Get merchant rules from db.json
- Call geminiService.generateOffer(context, rules)
- Save offer to db.json offers array with id, timestamp, context snapshot
- Return offer object

GET /api/offers:
- Return last 10 offers from db.json

POST /api/redeem  (body: { offerId, userId: "mia" }):
- Find offer in db.json
- Call qrService.generateQR(offerId, "mia")
- Save redemption record to db.json
- Return { success: true, qrDataUrl, cashbackAmount: (price * discount/100) }

POST /api/redeem/validate  (body: { token }):
- Find redemption by token in db.json
- Mark as used
- Return { valid: true, message: "Offer redeemed successfully" }

GET /api/merchant/dashboard:
- Calculate from db.json: offersToday, acceptRate, totalRevenueLift, avgDiscount
- Return metrics object

PUT /api/merchant/rules  (body: updated rules):
- Update merchant rules in db.json
- Return { success: true }

Fallbacks:
- Weather API fails → use { temp: 9, condition: "overcast", feelsLike: 6 }
- Gemini API fails → use hardcoded offer:
  { headline: "Cold outside? Come in.", 
    subtext: "Fresh batch brewed, quiet right now",
    discountPercent: 15, expiryMinutes: 20, 
    tone: "warm", ctaText: "Claim offer" }
- Redeem fails → still show a QR code generated locally
